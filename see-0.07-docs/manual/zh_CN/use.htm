1 SEE的界面

1.1 一般说明

   SEE是web应用，所以可以利用浏览器上的窗口功能：前进、后退、打印、另存为、将联接在新窗口中打开、文字的粘贴/复制等，这里不必说明了。
   用户的浏览器：

  • 应该支持javscript 1.0、cookie 1.0、和css 1.0
  • 显示SEE界面的理想显示分辨率为：1024 * 768

       以下是SEE界面的通用功能：

  • 在字段名上停留鼠标，可以显示相应的说明信息（如果定义了的话）。
  • 在鼠标移至联接/按钮/图形上，通常会变为一个手形，表明这是一个可点击的资源；但是对于没有这种特征的图片、有下划线的文字，也请试着点击。
  • 浏览器状态栏上显示：当前用户、当前项目、当前信息-操作、信息描述（如果定义了的话）。

    1.2 输入数据

       当添加或修改数据时：
  • 红色字段表示是关键字组，不能与已有数据相同。页面脚本自动判断。
  • 蓝色字段表示不能为空。页面脚本自动判断。
  • 复制数据时，应至少修改一个红色字段，使之保存为新数据，而不是覆盖原数据。页面脚本自动判断。
  • 对于时间或日期，最好点击旁边的“格式”按钮，来获得相应的格式。如果未按格式填写日期或时间的值，则排序、查询、统计时可能得不到合理的结果。

    1.3 列出数据

       执行插入、复制、修改、删除、显示、审计时，系统会先列出所有数据，在这些页面上：
  • 点击表头字段名字，可以排序，再次点击则以相反顺序排序。字段名旁若有“>”、“<”，表示按此字段升序或降序
  • 数据列表底部有翻页区域，显示：数据总数、当前序号，并允许用户设置每页数据个数（这个设置可以持续生效）、移至上页、下页或特定的页数。
  • 数据列表底部还有显示设置区域，显示当前：显示风格、显示字段、排序字段和排序方向。点击“显示设置”按钮，用户可以任意设置这些参数。参见SEE的显示设置的说明。
  • 数据列表底部还有查询设置区域，显示当前的查询条件。点击“查询设置”按钮，用户可以任意设置这些参数。参见SEE的查询设置的说明。
  • 如果信息结构中有定义，则数据列表上部还可能有颜色定义条。

       技巧1：利用排序和查询设置，可以方便地找到要操作的数据行。
       技巧2：在执行任何改变数据的操作前，先排序，则保存后的数据顺序就是当前排序结果。

       用户随时可以点击页面底部的“设置”按钮，来设置以下参数，它们立即生效、并可在本次登录中持续保持：

    界面方案 : 缺省方案

       其中：

  • 界面风格：指整个页面的图片和文字属性（字体/大小/颜色等）设计方案。目前提供两套。用户可以自己设计添加，添加的方式是“绿色的”，即只需将文件拷贝至相应的运行目录下即可，具体方法参考《源码说明》。
  • 界面方案：指如何在页面上按层次来显示功能按钮。目前只提供缺省的，用户可以自己修改、添加，即：超级管理员在使用界面上，点击顶部的“系统”按钮，再选择左面的“界面方案”按钮来管理界面方案，修改结果是即时生效的。
  • 数据显示风格：目前提供表单风格和表格风格两种。表格风格指每行显示一个数据，每列显示一个字段值。表单风格指逐个显示数据，每行显示一个字段值。
  • 每页数据个数：指显示多个数据时，每页显示的数据个数。
  • 缺省项目：指填写数据值时，可以缺省填写的项目标识。目前这个设置作用不大。
  • 语言切换按钮：SEE系统涉及2个语言编码：数据存储编码、和界面显示编码。目前SEE均以UTF-8的编码来保存数据，而界面显示语言可以随时切换。

       以上显示参数也可以在用户的个人信息中定义。不同的是：个人信息的设置参数在用户每次登录时被应用，而通过以上“设置”功能设置的参数只在本次登录生效；前者的修改下次登录生效，而后者的修改即时生效。

    2 SEE的信息处理

       通过选择3层功能菜单，用户可以访问到特定数据的处理页面，在页面上部有如下数据处理菜单：

    2.1 SEE的数据限制

       所谓数据限制指：用户输入的数据中不能包含的字符或字符串，它们将被自动过滤掉。

       目前SEE中还没有数据限制。

    2.2 SEE的特殊字符串

       所谓特殊字符串指：SEE用来实现信息处理的特定字符串。
       用户自己的数据中最好不要包含SEE的特殊字符串。

       目前SEE中包含如下特殊字符串：
    特殊字符串作用
    -99999999整型数的空值
    :c:e:q-c:e:q:在判断条件中作为"等于"解释。
    :c:n:e-c:n:e:在判断条件中作为"不等于"解释。
    :c:g:t-c:g:t:在判断条件中作为"大于"解释。
    :c:l:s-c:l:s:在判断条件中作为"小于"解释。
    :c:s:t-c:s:t:在判断条件中作为"开始于"解释。
    :c:e:d-c:e:d:在判断条件中作为"结束于"解释。
    :c:i:d-c:i:d:在判断条件中作为"包含"解释。
    :c:n:i-c:n:i:在判断条件中作为"不包含"解释。
    :m:s-m:s:多个数据的分隔符
    :c:i:s-c:i:s:条件判断的分隔符。
    :c:a:d-c:a:d:条件判断的“而且”。
    :a:a:d-a:a:d:统计分隔符。
    SEE-count-total统计总数的标记。用户不要用这个字段名
    SEE-count-other统计"其它值”的标记。用户不要用这个字段名
    SEE-count-null统计空值的标记。用户不要用这个字段名
    see_password_null口令字段的空值
    see_password_not_modify口令字段不修改
    :-:多个值的分隔符
    .h.p-h.p.hash名-值的分隔符
    :s:m-s:m:未知用途的分隔符
    :g:t-g:t:用户数据中">"符号将被保存为这个字符串。显示数据时显示为">"。
    :l:s-l:s:用户数据中"<"符号将被保存为这个字符串。显示数据时显示为"<"。
    :v:h-v:h:数据值的分隔符
    :a:d-a:d:用户数据中"&"符号将被保存为这个字符串。显示数据时显示为"&"。
    :r:t-r:t:用户数据中换行将被保存为这个字符串。显示数据时换行。
    _aaaa_矩阵字段名的分隔符。用户字段名中不要使用这个字串。
    SEE_matrix_value矩阵值字段名，用户字段不可取此字串。
    a:m:s-a:m:s审计值的分隔符

    2.3 添加数据

       在数据管理页面上，点击上部的“添加”页签，即进入添加数据的页面。
       新数据总是被加入当前数据列表的最后。
       注意事项参见    1.2节

    2.4 插入数据

       在数据管理页面上，点击上部的“插入”页签，即进入插入数据的页面。
       数据被加入到被选择行的前面。
       注意事项参见    1.2节和1.3节。

    2.5 修改数据

       在数据管理页面上，点击上部的“修改”页签，即进入修改数据的页面。
       注意事项参见    1.2节和1.3节。

    2.6 复制数据

       在数据管理页面上，点击上部的“复制”页签，即进入复制数据的页面。
       新数据的初始值就是被选择行的数据值。
       注意事项参见    1.2节和1.3节。

    2.7 导入数据

       在数据管理页面上，点击上部的“导入”页签，即进入导入数据的页面。
       在缺省权限模型中，只有超级管理员和数据管理员才能执行导入数据的操作。
       具体说明参见    “管理”第4节

    2.8 删除数据

       在数据管理页面上，点击上部的“删除”页签，即进入删除数据的页面。
       直接点击一行或行上的删除按钮，可以直接删除该行。用户也可以选择若干行来删除。
       在数据行下面，“本页全部”和“全不”按钮是用来选择本页数据行的，而“全部删除”则将删除所有数据（不只本页）。
       在缺省权限模型中，只有超级管理员和数据管理员才能执行删除数据的操作。
       其它注意事项参见    1.2节和1.3节。

    2.9 查询数据

       在数据管理页面上，点击上部的“查询”页签，即进入查询数据的页面。
       实际上，在数据处理其它页面上，也有查询设置的按钮，而查询页面比其它页面只是多了一个“导出”按钮。只有当数据不为空时，导出按钮才会显示。关于导出操作的说明，参见    第4节。
       其它注意事项参见1.2节和1.3节。
       关于查询设置，参见第6节。
       当数据是矩阵表时，页面上部会显示选择按钮，用户可以在主数据和矩阵数据之间切换。

    2.10 统计数据

       在数据管理页面上，点击上部的“统计”页签，即进入统计数据的页面。
       目前提供“计数”、“交叉计数”、“趋势”三种两种统计方法。
       统计产生的图片是jpg格式的临时文件，系统5分钟后自动删除。
       关于数据统计，参见    第7节。
       注意：只有当信息结构同时包含了时间/日期字段和整型/浮点字段时，趋势页签才会显示在统计页面上

    2.11 批量修改

       在数据管理页面上，点击上部的“批量修改”页签，即进入批量修改数据的页面。
       批量修改分为两步实现：第一步是选择需要修改的数据，第二步是填写要批量修改的数据值。

       矩阵字段可以被批量修改。
       单个关键字和口令字段不允许被批量修改。
       在缺省权限模型中，只有超级管理员和数据管理员才能执行批量修改的操作。

       在数据行下面，“本页全部”和“全不”按钮是用来选择本页数据行的，而“全部修改”按钮则将修改所有数据（不只本页）。
       其它注意事项参见    1.2节和1.3节。

    3 SEE的审计

    3.1 审计数据的查询

       在缺省权限模型中，所有一般用户都有权限查询审计数据。

       一般用户不能访问到审计记录的管理页面，而是通过其它信息管理页面上部的“审计”按钮来列出相应信息的审计记录。即：审计记录的查询是上下文相关的。
       例如，在“问题报告”的管理页面上，点击“审计”按钮，系统自动查询列出所有与“问题报告”有关的审计记录。

       若当前用户不是超级管理员，则与超级管理员有关的所有审计记录，将被系统自动过滤。

       在数据审计页面中，用户可以选择“按数据”还是“按时间”来显示审计记录。

    3.2 “按数据”来显示审计记录

    7 SEE的数据统计

    7.1 计数

       “计数”指：针对某个数据项，统计该项的不同值的分布个数和百分比：

  • 用户可以选择任何数据项来计数。
  • 用户可以输入要特别计数的值列表。若数据项值不在用户输入的值列表内，则作为“其它”值被计数。
  • 若用户不输入值列表，则SEE自动计数所有值。
  • 空值也会被计数。
  • 用户可以选择输出饼图和柱状图（jpg文件）。图中显示的数据值的个数是有限制的，但总显示值最大的几个。

    8 个人消息管理

       个人消息的管理原则为：

  • 用户只能查询自己发送和发给自己的消息、只能删除发给自己的消息。
  • 用户读过一条发给自己的消息后，系统自动标记其为“已读”。
  • 界面上提示新收到的消息个数。
  • 个人消息的内容不被审计。
