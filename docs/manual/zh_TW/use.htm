1 SEE的介面

1.1 一般說明

   SEE是web應用，所以可以利用瀏覽器上的窗口功能：前進、後退、列印、另存爲、將聯接在新視窗中打開、文字的粘貼/複製等，這裏不必說明了。
   用戶的瀏覽器：

  • 應該支援javscript 1.0、cookie 1.0、和css 1.0
  • 顯示SEE介面的理想顯示解析度爲：1024 * 768

       以下是SEE介面的通用功能：

  • 在欄位名上停留滑鼠，可以顯示相應的說明資訊（如果定義了的話）。
  • 在滑鼠移至聯接/按鈕/圖形上，通常會變爲一個手形，表明這是一個可點擊的資源；但是對於沒有這種特徵的圖片、有下劃線的文字，也請試著點擊。
  • 瀏覽器狀態欄上顯示：當前用戶、當前專案、當前資訊-操作、資訊描述（如果定義了的話）。

    1.2 輸入資料

       當添加或修改資料時：
  • 紅色欄位表示是關鍵字組，不能與已有資料相同。頁面腳本自動判斷。
  • 藍色欄位表示不能爲空。頁面腳本自動判斷。
  • 複製資料時，應至少修改一個紅色欄位，使之保存爲新資料，而不是覆蓋原資料。頁面腳本自動判斷。
  • 對於時間或日期，最好點擊旁邊的"格式"按鈕，來獲得相應的格式。如果未按格式填寫日期或時間的值，則排序、查詢、統計時可能得不到合理的結果。

    1.3 列出資料

       執行插入、複製、修改、刪除、顯示、審計時，系統會先列出所有資料，在這些頁面上：
  • 點擊表頭欄位名字，可以排序，再次點擊則以相反順序排序。欄位名旁若有">"、"<"，表示按此欄位昇冪或降冪
  • 資料列表底部有翻頁區域，顯示：資料總數、當前序號，並允許用戶設置每頁數據個數（這個設置可以持續生效）、移至上頁、下頁或特定的頁數。
  • 資料列表底部還有顯示設置區域，顯示當前：顯示風格、顯示欄位、排序欄位和排序方向。點擊"顯示設置"按鈕，用戶可以任意設置這些參數。參見SEE的顯示設置的說明。
  • 資料列表底部還有查詢設置區域，顯示當前的查詢條件。點擊"查詢設置"按鈕，用戶可以任意設置這些參數。參見SEE的查詢設置的說明。
  • 如果資訊結構中有定義，則資料列表上部還可能有顔色定義條。

       技巧1：利用排序和查詢設置，可以方便地找到要操作的資料行。
       技巧2：在執行任何改變資料的操作前，先排序，則保存後的資料順序就是當前排序結果。

    1.4 介面設置

       用戶隨時可以點擊頁面底部的"設置"按鈕，來設置以下參數，它們立即生效、並可在本次登錄中持續保持：

    設置

    介面風格 : 缺省風格 美麗的花
    介面方案 : 缺省方案
    資料顯示風格 : 表格 表單
    每頁數據個數 :
    缺省專案 :


    本系統安裝爲簡體中文，所以初始帳號和初始資料都是簡體中文的。
    語言選擇功能用來切換顯示不同語言的介面。

       其中：

  • 介面風格：指整個頁面的圖片和文字屬性（字體/大小/顔色等）設計方案。目前提供兩套。用戶可以自己設計添加，添加的方式是"綠色的"，即只需將文件拷貝至相應的運行目錄下即可，具體方法參考《源碼說明》。
  • 介面方案：指如何在頁面上按層次來顯示功能按鈕。目前只提供缺省的，用戶可以自己修改、添加，即：超級管理員在使用介面上，點擊頂部的"系統"按鈕，再選擇左面的"介面方案"按鈕來管理介面方案，修改結果是即時生效的。
  • 資料顯示風格：目前提供表單風格和表格風格兩種。表格風格指每行顯示一個資料，每列顯示一個欄位值。表單風格指逐個顯示資料，每行顯示一個欄位值。
  • 每頁數據個數：指顯示多個資料時，每頁顯示的資料個數。
  • 缺省專案：指填寫資料值時，可以缺省填寫的專案標識。目前這個設置作用不大。
  • 語言切換按鈕：SEE系統涉及2個語言編碼：資料存儲編碼、和介面顯示編碼。目前SEE均以UTF-8的編碼來保存資料，而介面顯示語言可以隨時切換。

       以上顯示參數也可以在用戶的個人資訊中定義。不同的是：個人資訊的設置參數在用戶每次登錄時被應用，而通過以上"設置"功能設置的參數只在本次登錄生效；前者的修改下次登錄生效，而後者的修改即時生效。

       頁面中部顯示各層功能按鈕需要打開的頁面。這個區域主要用來顯示資訊處理頁面。在資訊處理頁面的頂部，是資訊處理功能按鈕條。

       頁面底部是常用功能按鈕。包括：個人資訊、介面設置、會議記要、聯繫方式、問題報告、工作日志、用戶登錄、使用手冊、退出、聯繫作者。

       目前只提供缺省的介面方案，用戶可以自己修改、添加。具體的說明參見"定制"。

       除了頂部和底部按鈕，用戶只能看到自己有許可權使用的功能按鈕。

    2 SEE的資訊處理

       通過選擇3層功能功能表，用戶可以訪問到特定資料的處理頁面，在頁面上部有如下資料處理功能表：

    2.1 SEE的資料限制

       所謂資料限制指：用戶輸入的資料中不能包含的字元或字串，它們將被自動過濾掉。

       目前SEE中還沒有資料限制。

    2.2 SEE的特殊字串

       所謂特殊字串指：SEE用來實現資訊處理的特定字串。
       用戶自己的資料中最好不要包含SEE的特殊字串。

       目前SEE中包含如下特殊字串：

    2.3 添加資料

       在資料管理頁面上，點擊上部的"添加"頁簽，即進入添加資料的頁面。
       新資料總是被加入當前資料列表的最後。
       注意事項參見    1.2節

    2.4 插入資料

       在資料管理頁面上，點擊上部的"插入"頁簽，即進入插入資料的頁面。
       資料被加入到被選擇行的前面。
       注意事項參見    1.2節和1.3節。

    2.5 修改資料

       在資料管理頁面上，點擊上部的"修改"頁簽，即進入修改資料的頁面。
       注意事項參見    1.2節和1.3節。

    2.6 複製資料

       在資料管理頁面上，點擊上部的"複製"頁簽，即進入複製資料的頁面。
       新資料的初始值就是被選擇行的資料值。
       注意事項參見    1.2節和1.3節。

    2.7 導入資料

       在資料管理頁面上，點擊上部的"導入"頁簽，即進入導入資料的頁面。
       在缺省許可權模型中，只有超級管理員和資料管理員才能執行導入資料的操作。
       具體說明參見    "管理"第4節

    2.8 刪除資料

       在資料管理頁面上，點擊上部的"刪除"頁簽，即進入刪除資料的頁面。
       直接點擊一行或行上的刪除按鈕，可以直接刪除該行。用戶也可以選擇若干行來刪除。
       在資料行下面，"本頁全部"和"全不"按鈕是用來選擇本頁數據行的，而"全部刪除"則將刪除所有資料（不只本頁）。
       在缺省許可權模型中，只有超級管理員和資料管理員才能執行刪除資料的操作。
       其他注意事項參見    1.2節和1.3節。

    2.9 查詢資料

       在資料管理頁面上，點擊上部的"查詢"頁簽，即進入查詢資料的頁面。
       實際上，在資料處理其他頁面上，也有查詢設置的按鈕，而查詢頁面比其他頁面只是多了一個"導出"按鈕。只有當資料不爲空時，導出按鈕才會顯示。關於導出操作的說明，參見    第4節。
       其他注意事項參見1.2節和1.3節。
       關於查詢設置，參見第6節。
       當資料是矩陣表時，頁面上部會顯示選擇按鈕，用戶可以在主資料和矩陣資料之間切換。

    2.10 統計資料

       在資料管理頁面上，點擊上部的"統計"頁簽，即進入統計資料的頁面。
       目前提供"計數"、"交叉計數"、"趨勢"三種兩種統計方法。
       統計産生的圖片是jpg格式的暫存檔案，系統5分鐘後自動刪除。
       關於資料統計，參見    第7節。
       注意：只有當資訊結構同時包含了時間/日期欄位和整型/浮點欄位時，趨勢頁簽才會顯示在統計頁面上

    2.11 批量修改

       在資料管理頁面上，點擊上部的"批量修改"頁簽，即進入批量修改資料的頁面。
       批量修改分爲兩步實現：第一步是選擇需要修改的資料，第二步是填寫要批量修改的資料值。

       矩陣欄位可以被批量修改。
       單個關鍵字和口令欄位不允許被批量修改。
       在缺省許可權模型中，只有超級管理員和資料管理員才能執行批量修改的操作。

       在資料行下面，"本頁全部"和"全不"按鈕是用來選擇本頁數據行的，而"全部修改"按鈕則將修改所有資料（不只本頁）。
       其他注意事項參見    1.2節和1.3節。

    3 SEE的審計

    3.1 審計資料的查詢

       在缺省許可權模型中，所有一般用戶都有許可權查詢審計資料。

       一般用戶不能訪問到審計記錄的管理頁面，而是通過其他資訊管理頁面上部的"審計"按鈕來列出相應資訊的審計記錄。即：審計記錄的查詢是上下文相關的。
       例如，在"問題報告"的管理頁面上，點擊"審計"按鈕，系統自動查詢列出所有與"問題報告"有關的審計記錄。

       若當前用戶不是超級管理員，則與超級管理員有關的所有審計記錄，將被系統自動過濾。

       在資料審計頁面中，用戶可以選擇"按資料"還是"按時間"來顯示審計記錄。

    3.2 "按資料"來顯示審計記錄

       "按資料"來顯示審計記錄，是指：列出所有資料，針對每個資料，來查詢其所有的審計記錄。如下示例：

    問題報告 - 審計

    按資料 按時間
    狀態: 提交 已測試 已評審 被否定 已修正 已關閉 延遲 其他
    - 專案 類型 物件 狀態 嚴重性 優先順序 標題 指派給 最後修改時間
    - 錯誤 資料庫 已修正 高 高 lulu - 2004年4月28日 下午09時08分47秒
    - 錯誤 代碼 被否定 中等 中等 try - 2004年4月28日 下午08時00分44秒

    總計 : 2     當前序號 : 1 - 2 每頁 : 跳至頁 :

    風格:表格 項:[專案, 類型, 物件, 狀態, 嚴重性, 優先順序, 標題, 指派給, 最後修改時間]

       點擊資料行左面的"審計"按鈕，就會得到有關該行資料所有的審計記錄：

    問題報告 - 審計

    相關的審計記錄如下：
    標識 : 2004年4月28日 下午08時01分22秒-超級管理員
    資料 : 問題報告
    操作 : 修改
    關鍵字 : {配置編號=PR-try}
    原來的值 : {最後修改時間=2004年4月28日 下午08時00分11秒, 狀態=提交}
    新改的值 : {最後修改時間=2004年4月28日 下午08時00分44秒, 評議=不是問題, 狀態=被否定}
    誰 : 超級管理員
    何時 : 2004年4月28日 下午08時01分22秒
    何處 : 127.0.0.1
    結果 : 執行成功

    標識 : 2004年4月28日 下午08時00分30秒-超級管理員
    資料 : 問題報告

    7 SEE的資料統計

    7.1 計數

       "計數"指：針對某個資料項目，統計該項的不同值的分佈個數和百分比：

  • 用戶可以選擇任何資料項目來計數。
  • 用戶可以輸入要特別計數的值列表。若資料項目值不在用戶輸入的值列表內，則作爲"其他"值被計數。

    8 個人消息管理

       個人消息的管理原則爲：

  • 用戶只能查詢自己發送和發給自己的消息、只能刪除發給自己的消息。
  • 用戶讀過一條發給自己的消息後，系統自動標記其爲"已讀"。
  • 介面上提示新收到的消息個數。
  • 個人消息的內容不被審計。
