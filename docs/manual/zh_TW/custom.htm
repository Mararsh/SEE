1 介面方案的描述

   SEE的介面方案是在一個Xml表中描述和讀取的，超級管理員可以利用SEE提供的管理功能來修改它。

   方法是：超級管理員在SEE使用介面上，點擊頂部的"系統"按鈕，再選擇左面的"介面方案"按鈕來管理介面方案，修改結果是即時生效的。（別忘了刷新頁面）

   以下是SEE頂層功能表的定義：

介面方案 - 查詢

方案 : theme_default
名字 : see
子節點 : system_management:-:project_management:-:software_implementation:-:common_function
聯接 :

   以下是"系統"（system_management）的子功能表定義：

介面方案 - 查詢

方案 : theme_default
名字 : system_management
子節點 : init_config:-:interface_theme:-:language:-:user:-:role_definition:-:data_priviledge:-:data_audit:-:upload_management:-:login
聯接 :

   以下是"登錄"（login）的子功能表定義：

介面方案 - 查詢

方案 : theme_default
名字 : login
子節點 :
聯接 : frame_default?myframe=main&function=login

   所有定義都可以修改，只是定制者需要小心修改和刪除，不要刷新了頁面後發現什麽也用不了了。（如果真的發生了這樣的事，只需將源碼包裏缺省的介面方案文件"web\WEB-INF\data\value\see-value-interface_theme.xml"覆蓋你安裝目錄"WEB-INF\data\value\"下的此文件即可恢復）

2 添加/修改一個介面功能按鈕

   以下是添加一個介面功能按鈕的方法。
   例如，你想添加一個功能按鈕"tryfunc"，並且它是頂級功能表"常用"的子功能表項，可以執行如下步驟：

  • 以"超級管理員"賬號登錄。
  • 點擊介面頂部的"系統"按鈕，再選擇介面左面的"介面方案"按鈕,介面右面即是介面方案的管理頁面。
  • 點擊頁面上部的"添加"按鈕，進入添加資料的頁面。
  • 填寫它的"名字"域爲"tryfunc"，填寫它的"聯接"域爲"http://www.chong2.com"，保存。
  • 點擊頁面上部的"查詢"按鈕，進入查詢資料的頁面。點擊表頭"名字"，來排序查找，可以看到"tryfunc"已加入介面方案。
  • 點擊頁面上部的"修改"按鈕，進入修改頁面。
  • 找到"名字"爲"common_function"的資料行（可以點擊表頭"名字"來排序查找），點擊左側的"修改"按鈕，進入修改該資料的頁面。
  • 修改它的"子節點"域。資料是以":-:"分割的，在合適的位置加入"tryfunc"，保存。例如，改爲"meeting_summary:-:contact_way:-:tryfunc:-:share_resource:-:inform:-:manual"。（將以這個順序顯示子功能表項）
  • 點擊介面頂部的"常用"按鈕。看到什麽了？"tryfunc"按鈕出現在左面功能表條的第三行。點擊"tryfunc"按鈕，看到什麽了？

       你的新按鈕需要新圖片。SEE中圖片的存放路徑和命名約定，請參見《源碼說明》。

    3 刪除一個介面功能按鈕

       如果一個節點只有自己的定義，而不是任何其他節點的子節點，則它不會出現在介面上，因爲SEE是逐層顯示節點的。即：有效的節點應屬於see這棵樹。
       同時，如果一個節點只屬於某個節點，而沒有自己的定義，也不會顯示在介面上。即：有效的節點應有自己的定義。

       所以，刪除一個節點的方法，就是刪除它的定義，或者使它不出現在see這棵樹上。

    4 添加一個資訊結構

    4.1 添加介面功能按鈕

       添加一個資訊結構的方法，首先是添加一個介面功能按鈕。

       例如，你想添加一個資訊結構"tryinfo"，並且它是頂級功能表"常用"的子功能表項，可以執行如下步驟：

  • 以"超級管理員"賬號登錄。
  • 點擊介面頂部的"系統"按鈕，再選擇介面左面的"介面方案"按鈕,介面右面即是介面方案的管理頁面。
  • 點擊頁面上部的"複製"按鈕，進入複製資料的頁面。
  • 找到"名字"爲"change_request"的資料行（可以點擊表頭"名字"來排序查找），點擊左側的"複製"按鈕，進入複製該資料的頁面。
  • 修改它的"名字"域爲"tryinfo"，修改它的"聯接"域爲"data-manager?myset=tryinfo"。保存
  • 點擊頁面上部的"查詢"按鈕，進入查詢資料的頁面。點擊表頭"名字"，來排序查找，可以看到"tryinfo"已加入介面方案。
  • 點擊頁面上部的"修改"按鈕，進入修改頁面。
  • 找到"名字"爲"common_function"的資料行（可以點擊表頭"名字"來排序查找），點擊左側的"修改"按鈕，進入修改該資料的頁面。
  • 修改它的"子節點"域。資料是以":-:"分割的，在合適的位置加入"tryinfo"，保存。例如，改爲"meeting_summary:-:contact_way:-:tryinfo:-:share_resource:-:inform:-:manual"。（將以這個順序顯示子功能表項）
  • 點擊介面頂部的"常用"按鈕。看到什麽了？"tryinfo"按鈕出現在左面功能表條的第三行。點擊"tryinfo"按鈕，看到什麽了？

       你的新按鈕需要新圖片。SEE中圖片的存放路徑和命名約定，請參見《源碼說明》。

       由於採用xml保存定義，欄位名還有以下限制：不能以數位、"."、"_"開頭。頁面腳本將自動檢查這個限制。

       如果"翻譯值"選擇爲"是"，則缺省值和值約束都將被翻譯，即在保存資料時缺省值是經過翻譯的。否則即使顯示爲當前語言，保存的也是定義中的英文。
       （這只是對資訊結構的要求和處理，最終用戶在處理資訊資料時可以輸入任何文字，除了特殊字串，用戶輸入的資料不會被做任何處理。）

    4.4 定制的翻譯

       資訊結構的名、欄位的名、欄位缺省值和約束值都必須是英文的，而且不能有空格。而顯示給用戶、或寫入資料時不必受這些限制。
       系統的缺省資訊定義中所有要用到的翻譯，都已定義在程式中了。如果用戶添加、修改了資訊結構定義，就有可能用到翻譯列表之外的字串。
       此時可以利用SEE提供的翻譯功能，直接翻譯，即時生效。
       方法是：超級管理員點擊視窗上部的"系統"按鈕，再點擊左面的"語言翻譯"按鈕。

       例如，用戶定義了一個新的資訊結構：myinfo，其中有欄位：field1、feild2，一個欄位的約束值有：myv1,myv2,myv3。
       希望它們的簡體中文翻譯爲："我的資訊"、"我的欄位1"、"我的欄位2"、"我的值1"、"我的值2"、"我的值3"。
       希望它們的英文翻譯爲："my information"、"my field one"、"my field two"、"my value 1"、"my value 2"、"my value 3"。
       則添加如下簡體中文的翻譯

    簡體中文

    - 名字 < 值
    field1 我的欄位1
    field2 我的欄位2
    myinfo 我的資訊
    myv1 我的值1
    myv2 我的值2
    myv3 我的值3

       添加如下英文的翻譯

    英文

    - 名字 < 值
    field1 my field one
    field2 my field two
    myinfo my information
    myv1 my value 1
    myv2 my value 2
    myv3 my value 3

    5 預定義的資訊欄位

       描述完畢資訊結構，點擊頁面上部的"添加"按鈕和"查詢"按鈕，你就能看到自己定義的效果了。
       觀察缺省的資訊定義和使用情況，讀者應該大致明白它們的含義吧。
       作者建議用戶自己多試一試不同配置參數，能更直觀地瞭解SEE的定制能力。

       在這裏，給出SEE預定義資訊欄位的說明和顯示效果。

    5.1 自動賦值欄位

       如果一個資訊需要一個欄位來唯一區分各個資料記錄，而這個欄位的值實際沒有作用，則"自動賦值"類型的欄位可以滿足要求。

       定義的方法是：

  • 將欄位的"類型"選擇爲"自動賦值"

       在SEE中，自動賦值的欄位值由系統自動構造爲：

    創建時刻 + "-" + 當前用戶名

       顯示效果如下：
    標識 : 2004年6月5日 下午07時28分45秒-超級管理員

    5.2 文本欄位

       文本欄位是最常用的欄位類型。

       定義的方法是：

  • 將欄位的"類型"選擇爲"字串"
  • 如果要限制文本值的長度，則填寫"大小"。若"大小"爲空，則用戶輸入的任何長度的字串都有效。
  • 如果要設置文本輸入域的大小，則填寫"編輯寬度"。若"編輯寬度"爲空，則取缺省值（50）。
