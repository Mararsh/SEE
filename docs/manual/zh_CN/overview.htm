0 SEE是什么

   Software Engineering Environment(SEE)是一个信息处理框架。由sevlet+xml实现，通过浏览器使用。

   以下是SEE的几个示例界面：

登录 系统管理 项目管理 软件实现 常用功能
交叉饼图 交叉柱状图 比率柱状图 趋势图
个人消息 错误 “花”的界面

   安装SEE后，用户可以用“超级管理员”账号（口令为空）登录，试一试各种功能。
   在“工作周报”和“问题报告”中包含了一些示例数据，可以直接用来试用统计图表自动生成功能。

1 SEE简介

   Software Engineering Environment(SEE)企图提供软件开发的信息处理平台。
   SEE是一个免费项目，即作者将同时将提供所有源代码。
   它运行在任何支持servlet的环境下（作者只验证了win和linux环境），没用采用任何特定的实现技术，也不需要任何辅助程序。
   它的数据保存在XML中，不需要数据库。（将来的版本会支持数据存储的选项，这样用户可以选择mysql这样的免费数据库或者商业数据库，以提高处理速度）
   这意味着用户可以零成本运行它。（tomcat是免费的servlet运行环境）
   用户通过浏览器使用所有功能，因此信息共享非常方便。
   SEE目前支持简体中文、繁体中文和英语（只要有人翻译任何语言都可支持）。在运行时，用户可以随时切换不同语言的界面。用户亦可添加自己的文字翻译，即时生效。
   SEE主体是信息处理框架，其中所有信息均可动态定制，而且即时生效。这意味着用户可以随意随时修改、扩充自己的信息结构。SEE提供了信息属性的多种预定义选择，它们对应着多种预定义的信息处理模式。定制者的关注点在描述信息上，系统自动提供信息的处理功能。这是SEE的核心。SEE的绝大多数功能都是这样实现的。
   SEE缺省的信息定义，是作者对于软件工程信息需求的理解。
   虽然SEE目标是软件开发的信息处理，实际上，用户可以利用它设计处理任何类型的信息（例如图书登记、人事清单、财务报表等）。
   SEE的处理速度不差，20000条数据的读取写入不需1秒。目前排序速度是5000条9秒。（测试平台CPU为1.8G内存为256M）

3 SEE版本

3.1 SEE最新版本

   目前SEE的版本号是0.07，这是第七个公开发布的版本。这一版是为了纠正致命的错误而发版的。

   各版本的变化请参见《更新记录》。

   SEE的前几个版本都将重点实现功能框架，而不是设计信息结构。

   在版本1.0之前，用户最好不要实际运用SEE。SEE的设计和实现都不稳定。

3.2 新变化

3.2.1 新增的功能
   这一版新增的功能有：

  • 调整数据接口，准备接入数据库。
       这一版还未实现接入数据库。

  • 设计实现新的界面风格“美丽的花”。
       这些花都是老婆婆今年在公园里拍到的。
       老婆婆很喜欢这个叼花的小蜜蜂。这是老婆婆设计的第一个卡通形象。它将在你保存错误数据时出现。 ；）
       一个新界面风格就对应113*3张图片！

  • 实现“个人消息”功能。
       用户只能查询自己发送和发给自己的消息、只能删除发给自己的消息。
       用户读过一条发给自己的消息后，系统自动标记其为“已读”。
       界面上提示新收到的消息个数。
    3.2.2 增强的功能
       这一版增强的功能有：

  • 加强权限管理的实现。
       对于“全部删除”和“全部修改”，都是满足于“基本条件”的情况下执行，即用户只能全部删除或者全部修改自己可以访问的数据。
       只有有import权限的用户在导出数据时才可见password字段，其它情况均不可见。
       所有权限表重大修改！关键字改为“标题”。这样更有利于定义多个条件。
       实现权限表中“条件”的判断。现在可是设置成任何合法的SEE条件。

  • 显示图片的缩略图。
       在显示数据时，对于文件类型的字段值，若是图片（jpg/jpeg/gif/png），则显示其缩略图。

  • 使数据比较时与语言相关，这样各种语言的数据排序结果才正确。
       这个问题早就发现：中文排序结果没有按照拼音顺序。
    3.2.3 解决的问题
       这一版解决的问题有：（如此多的致命的、愚蠢的错误，作者为此沮丧了好一阵子~~~）

  • 一般用户不能访问“用户”表，因此很多数据都不能填写！！
       这是一个致命的错误！这就意味着以前的所有版本都无法实际使用！
       现在权限管理表重新定义了。

  • 新用户加入时，若不设置密码，则系统自己加上了密码！这样新用户就无法登陆！
       这又是一个致命的错误！

  • 在上一版（0.06）中，英文版的系统权限表居然是简体中文的！
       这样，英文版更是无法使用~~~
       这还是一个致命的错误！

  • 偏差处理里的“风险项”定义中引用“风险分析”表名错了。
       这就造成点击“选择”按钮时弹出了错误信息。

  • “口令”字段可以通过查询和导出看到。
       虽然是加密后的乱码，但也会造成隐患！
       现在是：只有有import权限的用户在导出数据时才可见password字段，其它情况均不可见。

  • 当关键字数据中包含引号时，页面会有错误，功能按钮（如修改、删除）无法使用！
       这是因为引号在页面数据引用中造成错误。

  • 界面语言变化时，审计记录的语言也变化了！
       这是不允许的。数据的语言是在安装时已确定，不应随界面语言变化！这个bug是因为用错了翻译方法。

    3.3 以后将实现的特点

  • 支持计算域的定义。
       计算域不是原始数据，其值不允许用户输入或修改，而是由其它数据根据即定公式自动得出。计算域常用于统计处理。

  • 提供更多的界面风格选择。
       目前只有一个缺省的界面风格。界面风格的添加是“绿色的”，即只需拷贝css和相应的图片到特定目录下，系统自己就会发现，因此可以独立提供给用户下载。

  • 实现数据存储方式的选择。
       目前数据是保存在xml文件中。将来的版本允许用户选择将数据存储在jdbc支持的任意数据库中，连接参数在系统安装时可配置。

  • 实现通用的工作流管理功能
       根据SEE自身的需求和框架，来实现实用的工作流机制。如果能兼顾国际标准更好。（难！）

  • 实现网络交流功能
       提供SEE自己的邮件管理、即时消息发送、论坛等功能。（工作量好大）

  • 设计实现软件项目管理的大多数信息处理。
       根据12207、cmm、国家标准的要求，分析设计信息结构。这些模型中有很多重复/类似的信息要求（与iso系列也有大量重复），系统只定义一套信息结构，用户可以切换不同的视角（界面方案）来进入这些模型。（难！）

  • 实现通用的文档生成功能。
       系统根据文档模板，由当前数据自动生成文档。系统提供符合12207、cmm、国家标准的文档模板，同时用户可以自定义文档模板。（难！）

       以上特征，需要作者一个版本一个版本地来逐步实现。也许这些需求会有修改、删除，也许会有更多的需求产生。

    3.4 上一版的特点

       截止上一版，SEE具有以下特色：

  • 免费。
       SEE自身免费，其运行环境均有免费的选择。免费指源代码可得。

  • 轻量的B/S模式。
       客户端只需通用的浏览器，服务器端只需servlet运行环境，SEE自身约6M（其中文档和图片为5.2M）。

  • 数据兼容。
       SEE中所有数据均是XML文件。而XML正是数据兼容的产物。

  • 跨平台。
       目前J2EE在多种操作系统上都有多种免费/商业产品。

  • 速度满足实际应用。
       测试的结果表明：xml读取写入速度几乎与数据量无关，当数据达到30000以上时，读取写入的耗时才一直处于1秒以上。
       SEE界面上有专门的按钮，用来测试SEE的数据处理速度。每个数据处理页面的右下角都会显示当前页面的响应速度。

  • 界面国际化和数据国际化。
       目前支持简体中文、英文、繁体中文。
       用户可以随时切换到不同的语言界面。
       不同语言的安装本版包含本地化的初始账号和初始数据。在使用时，预制数据自动保存为本地语言。

  • 用户语言翻译。
       现在用户可以直接使用SEE来添加自己的语言文字翻译。即时生效。
       这个功能对添加或修改信息结构的用户应该有用。

  • SEE的权限管理机制。
       设置超级管理员，区分系统角色和数据角色，可以分层定义系统角色、数据角色和用户的权限。（这种分层权限管理机制，参见使用指南的详细说明）
       安装后，系统中即有缺省的权限数据。缺省的权限模型基于这样的思路：最大限度地开放项目信息的管理能力，使项目信息对项目组成员完全透明。在功能上，每个成员都可以任意添加/修改/查询/审计项目的多数数据（删除权限只开放给数据管理员这个系统角色）。不必担心有人“捣乱”，因为所有操作行为、数据内容和执行结果都可以被系统自动记录。拥有充分的信息，任何人都可以监督任何人（超级管理员不可见，她/他是系统的构建者）。
       当然，如果用户需要采用其它权限模型，完全可以删除缺省数据、自己定义。

  • 用户配置功能。
       超级管理员可以配置整个系统的参数，其她/他用户可以设置实时生效的一些参数（界面语言、页面大小、数据显示风格、缺省项目等）。用户的个人信息中，还可以设置每次登录都会生效的参数。

  • 通用的信息结构定义功能。
       用户可以随时修改任何信息结构定义，即时生效。用户可以定义整个信息结构的属性（关键字组/关键字是否可修改/非空项/无效项/显示项/缺省颜色等）和各个数据字段的属性（数据类型/大小/编辑宽度/单位/缺省值/值约束/编辑控件类型等），系统根据信息的定义来自动提供信息的所有处理功能。这是SEE的核心。SEE的绝大多数数据和功能也是这样实现的。即：定义系统与使用系统一样，是通过运行SEE来实施的。

  • 通用的信息处理基本操作。
       即：添加、复制、插入、修改、删除。

  • 支持矩阵表。
       当一个信息结构包含多个交叉的属性时，使用矩阵表是最好的选择。

  • 实现批量修改功能。
       用户可以同时修改多个或者所有数据的某些字段值。矩阵字段可以被批量修改。
       在缺省权限模型中，只有超级管理员和数据管理员可以执行批量修改操作。

  • 通用的信息显示设置功能。
       用户可以任意设置信息显示的字段、排序、颜色、风格，并且可以保存、修改和删除这种显示设置。

  • 通用的信息查询设置功能。
       用户可以设置多个“而且”的条件关系，然后用“或者”的条件关系将它们连起来。条件算子有：等于、不等于、大于、小于、开始于、结束于、包含、不包含。系统依据数据类型来比较数据。
       用户可以保存、修改和删除查询设置。

  • 通用的信息统计分析功能。
       目前实现计数、交叉计数和趋势图三种类型。用户可以任意设置统计条件，系统自动生成统计报表（累计个数/百分率）、对比图形（饼图/柱形图）和时间-数据趋势图 。
       用户可以先设置查询条件、再统计过滤后数据。
       用户可以保存、修改和删除统计设置。

  • 数据导出功能。
       用户可以选择导出所有或者当前页面的数据，同时可以选择导出的格式：xml、txt或html。这样用户可以利用其它工具来做更细致的数据分析。同时利用显示设置功能，用户可以得到漂亮的查询报告。
       用户可以还选择导出文件的编码集，这样有利于将数据导入其它工具。

  • 数据导入功能。
       用户可以上传一个格式正确的xml文件来导入数据。
       在缺省的权限模型中，只有数据管理员和超级管理员可以导入数据。

  • 系统的审计功能。
       用户的数据操作可以被系统自动记录下来，包含：谁、何时、何地、什么数据、原来的字段值、新的字段值、操作结果等。这样，信息的数据变化可以被完整地跟踪下来。超级管理员可以设置需要审计的操作类型；用户可以按数据或者按时间来查看审计记录。这是项目监督的手段，也是数据恢复的一个途径。
       用户的登录行为也可以被审计。

  • 软件项目常用的信息结构。
       包括：项目跟踪（进度/规模/成本）、软件需求、软件配置项登记、软件评审、测试信息、更动申请、问题报告、会议记要等。

  • 网络应用的基本设施。
       包括：登录Cookie、口令加密、上传管理等。

    4 谁可能对SEE感兴趣

       以下理由可能会使读者对SEE感兴趣：

  • 你参与了一个软件项目，项目组需要处理并共享：问题报告、测试结果、更动申请、会议记要、文档登记、工作周报、工作日志之类的信息。那么SEE也许可以满足你的需要；如果需要修改缺省的信息定义，你不需要修改源码，而是运行SEE来配置。

  • 你所在的组织需要一个工具来处理并共享某类信息。运行SEE，添加一个新的信息结构，也许在5分钟内，你就可以实现你的想法。

  • 你想参考一个现成的信息处理软件系统。那么SEE的使用模式和设计实现，也许可能有一点帮助

  • 你希望观察一个现成的Sevelet应用。那么SEE也许可以是个例子。

  • 你需要直观了解各种软件工程模型（CMM/12207/国家标准）的信息要求。也许将来的SEE可以做出一点贡献。

    5 SEE不做什么

       SEE不打算成为包罗万象的软件工程信息处理平台。
       例如，对于配置管理，利用免费的cvs或者商业软件，也许更加有效。SEE至多可以提供信息接口，以供信息导入、导出、分析、查询。
       又如，对于项目规划和数据分析，使用唾手可得的现成工具（如project和excel），也许更为合理。SEE至多提供登记规划/分析结果的便利，以供项目组更容易地共享信息。
