1 界面方案的描述

   SEE的界面方案是在一个Xml表中描述和读取的，超级管理员可以利用SEE提供的管理功能来修改它。

   方法是：超级管理员在SEE使用界面上，点击顶部的“系统”按钮，再选择左面的“界面方案”按钮来管理界面方案，修改结果是即时生效的。（别忘了刷新页面）

   以下是SEE顶层菜单的定义：

界面方案 - 查询

方案 : theme_default
名字 : see
子节点 : system_management:-:project_management:-:software_implementation:-:common_function
联接 :

   以下是“系统”（system_management）的子菜单定义：

界面方案 - 查询

方案 : theme_default
名字 : system_management
子节点 : init_config:-:interface_theme:-:language:-:user:-:role_definition:-:data_priviledge:-:data_audit:-:upload_management:-:login
联接 :

   以下是“登录”（login）的子菜单定义：

界面方案 - 查询

方案 : theme_default
名字 : login
子节点 :
联接 : frame_default?myframe=main&function=login

   所有定义都可以修改，只是定制者需要小心修改和删除，不要刷新了页面后发现什么也用不了了。（如果真的发生了这样的事，只需将源码包里缺省的界面方案文件“web\WEB-INF\data\value\see-value-interface_theme.xml”覆盖你安装目录“WEB-INF\data\value\”下的此文件即可恢复）

2 添加/修改一个界面功能按钮

   以下是添加一个界面功能按钮的方法。
   例如，你想添加一个功能按钮“tryfunc”，并且它是顶级菜单“常用”的子菜单项，可以执行如下步骤：

  • 以“超级管理员”账号登录。
  • 点击界面顶部的“系统”按钮，再选择界面左面的“界面方案”按钮,界面右面即是界面方案的管理页面。
  • 点击页面上部的“添加”按钮，进入添加数据的页面。
  • 填写它的“名字”域为“tryfunc”，填写它的“联接”域为“http://www.chong2.com”，保存。
  • 点击页面上部的“查询”按钮，进入查询数据的页面。点击表头“名字”，来排序查找，可以看到“tryfunc”已加入界面方案。
  • 点击页面上部的“修改”按钮，进入修改页面。
  • 找到“名字”为“common_function”的数据行（可以点击表头“名字”来排序查找），点击左侧的“修改”按钮，进入修改该数据的页面。
  • 修改它的“子节点”域。数据是以“:-:”分割的，在合适的位置加入"tryfunc"，保存。例如，改为"meeting_summary:-:contact_way:-:tryfunc:-:share_resource:-:inform:-:manual"。（将以这个顺序显示子菜单项）
  • 点击界面顶部的“常用”按钮。看到什么了？"tryfunc"按钮出现在左面菜单条的第三行。点击"tryfunc"按钮，看到什么了？

       你的新按钮需要新图片。SEE中图片的存放路径和命名约定，请参见《源码说明》。

    3 删除一个界面功能按钮

       如果一个节点只有自己的定义，而不是任何其它节点的子节点，则它不会出现在界面上，因为SEE是逐层显示节点的。即：有效的节点应属于see这棵树。
       同时，如果一个节点只属于某个节点，而没有自己的定义，也不会显示在界面上。即：有效的节点应有自己的定义。

       所以，删除一个节点的方法，就是删除它的定义，或者使它不出现在see这棵树上。

       添加一个信息结构的方法，首先是添加一个界面功能按钮。

  • 以“超级管理员”账号登录。

  • 修改它的“名字”域为“tryinfo”，修改它的“联接”域为“data-manager?myset=tryinfo”。保存
  • 点击页面上部的“查询”按钮，进入查询数据的页面。点击表头“名字”，来排序查找，可以看到“tryinfo”已加入界面方案。
  • 点击页面上部的“修改”按钮，进入修改页面。
  • 找到“名字”为“common_function”的数据行（可以点击表头“名字”来排序查找），点击左侧的“修改”按钮，进入修改该数据的页面。

       （这只是对信息结构的要求和处理，最终用户在处理信息数据时可以输入任何文字，除了特殊字串，用户输入的数据不会被做任何处理。）

    4.4 定制的翻译

       信息结构的名、字段的名、字段缺省值和约束值都必须是英文的，而且不能有空格。而显示给用户、或写入数据时不必受这些限制。
       系统的缺省信息定义中所有要用到的翻译，都已定义在程序中了。如果用户添加、修改了信息结构定义，就有可能用到翻译列表之外的字符串。
       此时可以利用SEE提供的翻译功能，直接翻译，即时生效。
       方法是：超级管理员点击窗口上部的“系统”按钮，再点击左面的“语言翻译”按钮。

       例如，用户定义了一个新的信息结构：myinfo，其中有字段：field1、feild2，一个字段的约束值有：myv1,myv2,myv3。
       希望它们的简体中文翻译为："我的信息"、“我的字段1”、“我的字段2”、“我的值1”、“我的值2”、“我的值3”。
       希望它们的英文翻译为："my information"、“my field one”、“my field two”、“my value 1”、“my value 2”、“my value 3”。
       则添加如下简体中文的翻译

    简体中文

    - 名字 < 值
    field1 我的字段1
    field2 我的字段2
    myinfo 我的信息
    myv1 我的值1
    myv2 我的值2
    myv3 我的值3

       添加如下英文的翻译

    英文

    - 名字 < 值
    field1 my field one
    field2 my field two
    myinfo my information
    myv1 my value 1
    myv2 my value 2
    myv3 my value 3

    5 预定义的信息字段

       描述完毕信息结构，点击页面上部的“添加”按钮和“查询”按钮，你就能看到自己定义的效果了。
       观察缺省的信息定义和使用情况，读者应该大致明白它们的含义吧。
       作者建议用户自己多试一试不同配置参数，能更直观地了解SEE的定制能力。

       在这里，给出SEE预定义信息字段的说明和显示效果。

    5.1 自动赋值字段

       如果一个信息需要一个字段来唯一区分各个数据记录，而这个字段的值实际没有作用，则“自动赋值”类型的字段可以满足要求。

       定义的方法是：

  • 将字段的“类型”选择为“自动赋值”

       在SEE中，自动赋值的字段值由系统自动构造为：

    创建时刻 + “-” + 当前用户名

       显示效果如下：
    标识 : 2004年6月5日 下午07时28分45秒-超级管理员

    5.2 文本字段

       文本字段是最常用的字段类型。

       定义的方法是：

  • 将字段的“类型”选择为“字符串”
  • 如果要限制文本值的长度，则填写“大小”。若“大小”为空，则用户输入的任何长度的字符串都有效。
  • 如果要设置文本输入域的大小，则填写“编辑宽度”。若“编辑宽度”为空，则取缺省值（50）。
  • 可以填写“缺省值”，使得创建新数据记录时系统自动填写一个值。技巧：若字段值具有某种特定的前缀、后缀、格式、或者特殊字符，可以设置为缺省值。
  • 如果"翻译值"选择为“是”，则缺省值将被翻译，即在保存数据时缺省值是经过翻译的。

       显示效果如下：
    文本 :

    5.3 日期字段

       日期字段允许用户输入和修改。

       定义的方法是：

  • 将字段的“类型”选择为“日期”
  • “缺省值”可以生效，但是不建议填写。
  • 不要设置“值约束类型”

       使用的方法是：

  • 系统将显示一个文本填写域和一个“格式”按钮，点击“格式”按钮后，系统自动填写格式正确的当天值
  • 用户可以修改。注意要保证格式正确，否则排序时得不到合理结果。

       显示效果如下：
    日期 :

       定义的方法是：

  • 用户可以修改。注意要保证格式正确，否则排序时得不到合理结果。
